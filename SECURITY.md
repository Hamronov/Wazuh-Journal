# Безопасность

Не добавляйте в Git файлы `.env.local`, приватные ключи, пароли или сертификаты. Используйте `.env.example` только как шаблон и храните значения переменных окружения в секретном хранилище среды выполнения.

В production обязательно задайте `AUTH_SESSION_SECRET` случайной строкой длиной не менее 32 символов. Wazuh, LDAP и OpenAI credentials используются только на сервере и не должны попадать в клиентский код или логи.

Если секрет был раскрыт, немедленно отзовите его у соответствующего провайдера и только после этого удаляйте его из файлов или истории.
